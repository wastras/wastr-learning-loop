# Monthly Product Review — Ritual Guide

> **Cadence:** last Friday of the month, 90 min.
> **Output:** updated `pilot-learnings.md`, updated `roadmap-now-next-later.md`, and any new `[DECISION]` issues.

## Agenda (90 min)

| Time | Topic |
|---|---|
| 0–10 | Recap last month's bets and their outcomes |
| 10–25 | Walk through `[SIGNAL]` issues since last review — cluster into themes |
| 25–40 | Walk through `[EXP]` results — validated / invalidated |
| 40–55 | Update `pilot-learnings.md` with synthesis |
| 55–70 | Update `roadmap-now-next-later.md` (promote, drop, park) |
| 70–85 | Open / refine next month's bets (`02_product_bet` issues) |
| 85–90 | Capture any architectural or business decisions as `[DECISION]` issues |

## Inputs (gather before the meeting)

- All issues opened in the last 30 days, grouped by label
- Current metric values vs north-star targets
- All experiment results since last review
- Sales / pilot pipeline status

## Definition of Done for the Review

- [ ] `pilot-learnings.md` updated and committed
- [ ] `roadmap-now-next-later.md` updated and committed
- [ ] At least one `[DECISION]` issue filed if direction changed
- [ ] Next month's top 3 bets exist as issues with success criteria
- [ ] Any `learning: revisit` items either re-opened or archived
